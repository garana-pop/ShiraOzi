## ストーリー

### 冒頭

目が覚めると、おじいちゃんが心配そうにこちらを見ていた。  
なぜか私の家にいて、当たり前のように料理をし始める。  
一緒に料理を食べながら、おっさんの人となりが分かっていく。  
タバコが、二人の共通点。

### エンディング

日記をヒントにエレベーターのパスワードを解き、現実に戻る。  
描かれていた場所は天国のようなところで、二人とも現実世界ではお風呂場で死亡していた。  
しかし主人公の女は婚約者に振られ自殺しようとしたが、まだ仮死状態であり——おっさんとの出会いが、彼女を現実へ引き戻す。

### 細かい設定

- **仮死状態になった日時**：2022年8月13日（お盆開始時期）18:01
- **18:01の理由**：主人公は"しなない"（死なない）= 数字で「4771」→ 4+7+7=18、残り1 → `18:01`
- **最終パスワード**：`4771`（私は死なない）
- スマホの時間は永遠に `18:01` のまま
- **「お嬢」の由来**：最初は「お嬢さん」と呼んでいたが、主人公が年上に"さん"をつけられるのを嫌がったため

## ゲームプレイ

### 基本ループ

1. **探索** — 部屋の中のオブジェクトをクリックして調べる
2. **インタラクション** — ホバーでカーソル変化＋SE。クリックで説明表示またはアイテム取得
3. **対話** — 同居人のおっさんに話しかける。アイテム所持中は関連した思い出話が始まる
4. **進行** — インタラクション完了で章が進み、日記が更新される
5. **パズル** — 日記のヒントを元に最終パスワードを解いてエンディングへ

### 操作方法

| 操作 | 内容 |
|---|---|
| 左クリック | オブジェクトへのインタラクション／ダイアログ送り／メニュー操作 |
| ホバー | インタラクション可能オブジェクトでカーソル変化＋SE |

## 主要スクリプト

### Core

#### `GameState.cs`（ScriptableObject）

ゲームの進行状態を一元管理する。

| フィールド | 型 | 説明 |
|---|---|---|
| `currentChapter` | `int` | 現在の章番号 |
| `heldItemID` | `string` | 現在アクティブなアイテムID |
| `acquiredItems` | `List<ItemData>` | 取得済みアイテム一覧 |
| `activeItem` | `ItemData` | 現在選択中のアイテム |
| `unlockedDiaryEntries` | `List<string>` | 解放済み日記エントリID一覧 |

#### `DialogueManager.cs`

ダイアログの再生・送り・ローカライズ対応を管理するシングルトン。  
`StartDialogue(DialogueEntry)` → `AdvanceDialogue()` → `EndDialogue()` の流れで動作。

#### `ChapterManager.cs`

章の切り替えを管理するシングルトン。`NextChapter()` / `SetChapter(int)` で操作。

### Interaction

#### `InteractableObject.cs`

シーン内のオブジェクトにアタッチする。ホバー・クリックイベントを `UnityEvent` で公開。  
`CursorManager` / `SoundManager` と連携してフィードバックを提供。

#### `PasswordPuzzle.cs`

エレベーターの最終パスワード入力を処理する。正解は `4771`（インスペクターで変更可）。

#### `NPCInteraction.cs`

NPCクリック時に、所持アイテムに応じたダイアログを再生する。  
`DialogueLayoutSettings` コンポーネントがあれば自動的にレイアウトを適用。

### UI

#### `UIManager.cs`

UIの表示制御の司令塔となるシングルトン。  
ダイアログパネル・アイテムパネル・設定パネルの表示管理とレイアウト変更を担う。

#### `DiaryManager.cs`

`GameState.unlockedDiaryEntries` に基づいて日記エントリを動的生成・表示する。

#### `InventoryUI.cs`

アイテムパネルのトグル表示と、取得済みアイテムの一覧表示を管理する。  
ESCキーで閉じることも可能。

## コーディングルール

### 命名規則

| 種類 | 規則 | 例 |
|---|---|---|
| パブリックフィールド・プロパティ | PascalCase | `PlayerName` |
| プライベートフィールド | camelCase | `moveSpeed` |
| 定数 | UPPER_SNAKE_CASE | `MAX_HEALTH` |
| SerializeField | camelCase | `targetObject` |

### コメント規則

- 変数コメントは宣言の横に記述
- 処理コメントは処理の1行上に記述
- `//` の後に半角スペース1つを入れる
- メソッドはドキュメンテーションコメント（`///`）で記述

```csharp
/// <summary>
/// アイテムをインベントリに追加する。
/// </summary>
/// <param name="item">追加するアイテムデータ</param>
public void AddItem(ItemData item) { ... }

private int playerHP;   // プレイヤーのHP (0～100)
```

### その他

- マジックナンバーは定数化する
- コンポーネント参照は `[SerializeField]` を使用
- デバッグログにはクラス名を含める：`Debug.Log($"{nameof(ClassName)}: メッセージ");`
