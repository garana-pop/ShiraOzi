# プロジェクト概要
- ゲームタイトル: ShiraOzi
- コンセプト: 29歳の女性と65歳のおっさんがワンルームで同居し、部屋のアイテムを通じて過去を紐解いていく2Dクリック型探索アドベンチャー。
- プレイヤー: シングルプレイヤー
- 参考ゲーム: ファミレスを享受せよ
- トーン / アート方向性: ギャグ漫画のようなセリフ回しと、背景にあるエモーショナルで天国のようなテーマ。
- ターゲットプラットフォーム: PC (Standalone Windows 64)
- 画面方向 / 解像度: 横画面 1920x1080
- レンダリングパイプライン: UniversalRP (2D)

# ゲームメカニクス
## コアゲームプレイループ
1. **探索**: プレイヤーは部屋の中の様々なオブジェクトをクリックします。
2. **インタラクション**: オブジェクトにマウスホバーするとカーソルが変化し、SEが鳴ります。クリックすると説明が表示されるか、そのアイテムを「持つ」ことができます。
3. **対話**: 同居人である「おっさん」に話しかけます。アイテムを持っている場合、そのアイテムに関連した思い出話が始まります。
4. **進行**: インタラクションを完了することで章が進み、日記が更新されます。
5. **パズル**: 日記に集めたヒントを元に、最終的なパスワードパズルを解いて現実世界への帰還を目指します。

## 操作方法と入力
- **マウスのみ**:
  - **左クリック**: オブジェクトへのインタラクション、ダイアログの送り、メニュー操作。
  - **ホバー**: インタラクション可能なオブジェクト上での視覚・聴覚的フィードバック。
- **入力システム**: New Input System（`Pointer`および`Click`アクションを使用）。

# UI
- **ダイアログボックス**: 画面下部に固定。話者の名前とローカライズされたテキストを表示。
- **アイテム表示**: 現在「持っている」アイテムを画面の隅に表示。
- **設定メニュー**: フルスクリーン/ウィンドウの切り替え、解像度選択、言語選択（日本語、英語、簡体字、繁体字）。
- **日記画面**: 収集した日記のエントリやヒントを確認できる専用画面。

# 主要アセットとコンテキスト
- `GameState.asset` (ScriptableObject): 現在の章、保持アイテム、日記の状態を管理。
- `DialogueEntry.asset` (ScriptableObject): ローカライズキーと話者データを含む。
- `InteractableObject.cs`: シーン内のスプライトにアタッチするコンポーネント。
- `Localization Tables`: 4言語分の文字列テーブル。
- `SoundManager.cs`: ホバーSEやダイアログSEを管理。

# 実装ステップ
1. **コア基盤の構築**
   - プレイヤーの進行状況を保存する `GameState` ScriptableObject の作成。
   - マウス操作用の Input Action Map のセットアップ。
   - [依存関係: なし]

2. **インタラクションシステム**
   - `IPointerEnterHandler` 等を利用した `InteractableObject` の実装。
   - ホバー時のカーソル変化を管理する `CursorManager` の作成。
   - ホバーSEの実装。
   - [依存関係: ステップ 1]

3. **ダイアログ・アイテムシステム**
   - `DialogueEntry` を読み込む `DialogueManager` の作成。
   - アイテムをクリックした際に `GameState` を更新する「アイテム保持」機能の実装。
   - おっさんに話しかけた際、保持アイテムをチェックして対応する会話を再生するロジック。
   - [依存関係: ステップ 2]

4. **ローカライズ設定**
   - Unity Localization パッケージのインストールと設定。
   - 日本語、英語、簡体字、繁体字のロケール作成。
   - 初期ダイアログキーの入力。
   - [依存関係: ステップ 3]

5. **UIと設定メニュー**
   - ダイアログボックスと保持アイテム表示を含むHUDの作成。
   - 設定メニュー（フルスクリーン切り替え、解像度、言語）の実装。
   - [依存関係: ステップ 4]

6. **日記と進行管理**
   - `GameState` のフラグに基づいて内容が解放される日記システム。
   - 全6章の切り替えを管理するチャプターマネージャー。
   - [依存関係: ステップ 5]

7. **最終パズルとエンディング**
   - エレベーターのパスワード入力システムの実装。
   - ストーリー概要に基づいたエンディング演出のセットアップ。
   - [依存関係: ステップ 6]

# 検証とテスト
- **入力テスト**: ホバー時にカーソルが変わり、SEが鳴るか確認。
- **ダイアログテスト**: 4言語すべてでテキストが正しく表示されるか確認。
- **アイテムテスト**: アイテム保持状態でおっさんに話しかけた際、正しい会話が再生されるか確認。
- **解像度テスト**: ウィンドウ/フルスクリーンの切り替えが正しく動作するか確認。
- **状態保存テスト**: 日記の内容が正しく記録・再現されるか確認。
